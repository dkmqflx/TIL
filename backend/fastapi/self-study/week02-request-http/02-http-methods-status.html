HTTP 메서드 & 상태 코드
API의 동사와 결과 보고

메서드는 "무엇을 할지"를, 상태 코드는 "어떻게 됐는지"를 말합니다. 이 둘을 정확히 쓰는 것만으로 API의 품질이 절반은 결정됩니다.

“

메서드는 동사(GET 읽기 · POST 만들기 · PATCH 고치기 · DELETE 지우기), 상태 코드는 한 줄 결과 보고(2xx 성공 · 4xx 네 잘못 · 5xx 내 잘못)다. 생성은 201, 본문 없는 삭제는 204.

  • 주요 HTTP 메서드의 의미와 멱등성(idempotent)을 이해한다
  • 상태 코드를 범위별로 알고, 상황에 맞게 고른다
  • FastAPI에서 상태 코드를 지정하는 법을 익힌다

1메서드 — API의 동사들

메서드의미안전?멱등?본문
GET조회✅ 안전✅ 멱등없음
POST생성❌❌ 비멱등있음
PUT전체 교체❌✅ 멱등있음
PATCH부분 수정❌△ 보통 비멱등있음
DELETE삭제❌✅ 멱등보통 없음

"안전(safe)"과 "멱등(idempotent)"이 뭔가요?

안전 = 서버 상태를 바꾸지 않음(읽기 전용). 멱등 = 같은 요청을 여러 번 보내도 결과가 같음.

🛗 비유 — 엘리베이터 vs 초인종

멱등한 요청(PUT·DELETE)은 엘리베이터 호출 버튼입니다. 열 번 눌러도 결과는 "엘리베이터가 온다" 하나뿐이죠. 반면 비멱등한 POST는 "택배 주문하기" 버튼 — 열 번 누르면 택배가 열 개 옵니다! 그래서 결제·주문 같은 POST는 중복 요청 방지가 중요합니다.

💡 왜 면접에서 멱등성을 물을까

네트워크가 끊겨 클라이언트가 요청을 재시도할 때, 멱등한 메서드는 안심하고 다시 보낼 수 있습니다. 비멱등한 POST는 중복 생성 위험이 있어 별도 장치(고유 키, 멱등성 키)가 필요하죠. 이 차이를 아는지가 곧 "HTTP를 제대로 이해했는가"의 척도입니다.

2상태 코드 — 결과 한 줄 보고

응답의 첫 줄에 실리는 세 자리 숫자입니다. 백 자리만 봐도 큰 그림을 알 수 있습니다.

코드이름언제
200OK일반 성공 (조회·수정)
201Created새 자원 생성 성공 (POST)
204No Content성공했지만 돌려줄 본문 없음 (DELETE)
400Bad Request요청 형식 자체가 잘못
401Unauthorized인증 안 됨 (로그인 필요)
403Forbidden인증은 됐지만 권한 없음
404Not Found자원이 없음
422UnprocessableFastAPI의 검증 실패 (타입·필수 등)
500Server Error서버 코드에서 처리 못 한 예외
⚠️ 401 vs 403 헷갈리지 않기

401은 "너 누구냐?"(인증 안 됨 — 로그인하라), 403은 "너인 건 알겠는데 출입 금지"(권한 없음)입니다. 로그인 안 한 사용자 → 401, 일반 회원이 관리자 페이지 접근 → 403.

3FastAPI에서 상태 코드 지정하기

FastAPI는 성공 시 기본 200을 돌려줍니다. 생성은 201, 삭제는 204로 바꾸려면 데코레이터에 status_code를 줍니다.

main.py
from fastapi import FastAPI, status

app = FastAPI()

@app.post("/todos", status_code=status.HTTP_201_CREATED)
def create(todo: CreateTodo):
    return todo  # → 201 Created

@app.delete("/todos/{id}", status_code=status.HTTP_204_NO_CONTENT)
def delete(id: int):
    return  # 204는 본문이 없어야 한다 → None 반환
💡 왜 status.HTTP_201_CREATED를 쓰나

그냥 201 숫자를 써도 되지만, status 모듈의 상수를 쓰면 오타 방지 + 자동완성 + 가독성이 좋아집니다. 201이 무슨 뜻인지 코드만 봐도 읽히죠.

Nest NestJS
todos.controller.ts
@Post()
@HttpCode(201)
create(@Body() dto) {}

NestJS는 POST 기본이 이미 201.

Fast FastAPI
main.py
@app.post(
  "/todos",
  status_code=201,
)
def create(dto): ...

FastAPI는 기본 200이라 명시 필요.

4✅ 셀프 체크

Q PUT과 PATCH는 멱등성이 어떻게 다른가요?

PUT은 자원 전체를 보낸 값으로 교체하므로 같은 요청을 반복해도 결과가 같아 멱등합니다. PATCH는 부분 수정이라 보통 멱등으로 보지만, "조회수 +1" 같은 상대적 변경을 담으면 비멱등이 됩니다 — 구현에 달렸습니다.

Q 삭제 성공에 200이 아니라 204를 권하는 이유는?

삭제 후 돌려줄 데이터가 보통 없기 때문입니다. 204 No Content는 "성공했고 본문은 비어 있다"를 정확히 표현합니다. 단, 삭제된 객체를 응답으로 돌려주고 싶다면 200 + 본문도 유효한 선택입니다.

Q 요청 JSON에 필수 필드가 빠지면 몇 번이 응답되나요?

422 Unprocessable Entity입니다. FastAPI/Pydantic의 검증 실패는 400이 아니라 422를 씁니다. 어떤 필드가 왜 틀렸는지 상세 내역도 함께 돌려줍니다.
