response_model
나가는 데이터도 검사한다

들어오는 요청만큼 나가는 응답도 중요합니다. response_model은 "이 API는 정확히 이 모양만 돌려준다"를 선언해, 실수로 비밀번호 같은 민감 정보가 새어나가는 것을 막습니다.

“

요청 검사가 입국 심사라면, response_model은 출국 검색대다. 정해둔 모양에 없는 필드(예: password)는 밖으로 못 나간다. 입력 모델과 출력 모델을 분리하는 것이 실무의 기본기다.

  • response_model로 응답의 형태를 선언하고 필터링한다
  • 입력용·출력용 모델을 분리하는 이유와 패턴을 안다
  • JSONResponse 등 Response 클래스로 응답을 세밀하게 제어한다

1왜 응답에도 모델이 필요한가

회원 정보를 돌려주는 API를 생각해봅시다. DB의 User 객체에는 hashed_password가 들어 있습니다. 아무 생각 없이 그 객체를 반환하면? 비밀번호 해시가 그대로 클라이언트에 노출됩니다. 끔찍하죠.

🛃 비유 — 출국 검색대

입국 심사대(요청 검증)만 있고 출국 검색대가 없으면, 가져가면 안 되는 물건(민감 정보)이 그대로 빠져나갑니다. response_model은 "이 게이트로는 이 목록에 있는 것만 통과"라고 못 박는 출국 검색대입니다. 목록에 없는 hashed_password는 자동으로 걸러집니다.

2입력 모델 ≠ 출력 모델

실무의 핵심 패턴: 받을 때(Create)와 돌려줄 때(Public) 모델을 나눕니다. 회원가입을 예로 보죠.

schemas.py
from pydantic import BaseModel

# 받을 때: password 포함
class UserCreate(BaseModel):
    username: str
    email: str
    password: str

# 돌려줄 때: password 없음! (id는 추가)
class UserPublic(BaseModel):
    id: int
    username: str
    email: str

이제 라우트에 response_model=UserPublic을 지정합니다. 함수가 비밀번호를 포함한 객체를 반환해도, FastAPI가 UserPublic 모양으로 걸러서 내보냅니다.

main.py
@app.post("/users", response_model=UserPublic, status_code=201)
def create_user(user: UserCreate):
    saved = {"id": 1, **user.model_dump()}  # password 포함된 dict
    return saved   # 그래도 응답엔 password가 안 나간다! 🎉

출력 모델을 통째로 선언.

💡 response_model의 3가지 이득

① 보안 — 민감 필드 자동 제거 · ② 문서 — Swagger에 정확한 응답 스키마 표시 · ③ 검증 — 응답조차 모델에 맞는지 확인(잘못된 응답이면 서버 에러로 잡힘). 입력 검증만큼 출력 계약도 명확해집니다.

3유용한 옵션 몇 가지

옵션효과
response_model_exclude_unset=True설정 안 한(기본값인) 필드는 응답에서 생략
response_model_exclude={"email"}특정 필드 제외
response_model_include={"id","username"}특정 필드만 포함

4Response 클래스 — 응답을 직접 빚기

대부분은 dict나 모델을 반환하면 FastAPI가 알아서 JSON으로 만들어주지만, 가끔 응답 형식·헤더·상태를 직접 제어해야 할 때가 있습니다. 그때 Response 클래스를 씁니다.

클래스용도
JSONResponse기본 JSON 응답 (직접 상태·헤더 제어)
ORJSONResponseorjson 기반 초고속 JSON (대용량에 유리)
PlainTextResponse순수 텍스트
RedirectResponse리다이렉트(3xx)
StreamingResponse스트리밍(대용량 파일·실시간)
main.py
from fastapi.responses import JSONResponse, RedirectResponse

@app.get("/custom")
def custom():
    return JSONResponse(
        content={"ok": True},
        status_code=200,
        headers={"X-Custom": "hello"},
    )

@app.get("/old-path")
def moved():
    return RedirectResponse("/new-path", status_code=307)
⚠️ Response를 직접 반환하면 response_model이 적용 안 된다

JSONResponse(...)처럼 완성된 Response 객체를 반환하면, FastAPI는 "이미 다 만들었구나" 하고 response_model 필터링·문서 생성을 건너뜁니다. 평소엔 모델/dict를 반환하고, 정말 세밀한 제어가 필요할 때만 Response 클래스를 쓰세요.

복습

5✅ 셀프 체크

Q 함수가 password를 포함해 반환했는데도 응답에 안 나오는 이유는?

라우트에 response_model=UserPublic을 지정했기 때문입니다. FastAPI가 반환값을 UserPublic 모양으로 다시 직렬화하면서, 그 모델에 없는 password 필드를 자동으로 제거합니다.

Q 왜 입력 모델(UserCreate)과 출력 모델(UserPublic)을 나누나요?

받는 데이터와 돌려주는 데이터의 형태가 다르기 때문입니다. 입력엔 password가 필요하지만 출력엔 없어야 하고, 출력엔 서버가 만든 id가 추가됩니다. 분리하면 보안·문서·유지보수가 모두 명확해집니다.

Q JSONResponse를 직접 반환할 때 주의점은?

response_model 필터링과 자동 문서 생성이 적용되지 않습니다. 완성된 Response를 반환하면 FastAPI가 후처리를 건너뛰기 때문입니다. 꼭 필요한 경우에만 쓰고, 평소엔 모델/dict 반환을 권장합니다.
