APIRouter
도메인별로 코드를 나누기

모든 라우트를 main.py 하나에 욱여넣으면 금방 수백 줄이 됩니다. APIRouter로 업무 도메인별 파일을 나누는 법 — NestJS의 Module 구조에 정확히 대응합니다.

“

APIRouter는 라우트들을 도메인 단위로 묶는 미니 앱이다. todos 라우터, users 라우터를 따로 만들고 메인 앱에 include_router로 꽂는다. NestJS의 Module/Controller 분리와 같은 발상.

  • APIRouter로 라우트를 도메인별 파일로 분리한다
  • prefix와 tags로 공통 경로·문서 그룹을 지정한다
  • 실무에서 통하는 폴더 구조를 잡는다

1왜 나누는가

🏢 비유 — 부서로 나눈 회사

직원 한 명이 회계·인사·영업을 다 하면 회사가 굴러갈까요? 규모가 커지면 부서(도메인)로 나눕니다. APIRouter는 "할 일 부서(todos)", "회원 부서(users)"처럼 라우트를 갈라 담는 부서 칸막이입니다. 메인 앱은 각 부서를 한데 모으는 본사 역할만 합니다.

2라우터 만들고 꽂기

① 도메인별 파일에서 APIRouter를 만들고 라우트를 등록한 뒤, ② main.py에서 include_router로 합칩니다.

routers/todos.py
from fastapi import APIRouter

router = APIRouter(prefix="/todos", tags=["todos"])

@router.get("")          # 실제 경로: GET /todos
def list_todos():
    return [...]

@router.get("/{id}")      # 실제 경로: GET /todos/{id}
def get_todo(id: int):
    return {...}
main.py
from fastapi import FastAPI
from routers import todos, users

app = FastAPI()

app.include_router(todos.router)   # 할 일 부서 연결
app.include_router(users.router)   # 회원 부서 연결
💡 prefix와 tags의 역할

prefix="/todos"를 주면 그 라우터의 모든 경로 앞에 /todos가 자동으로 붙습니다 — 라우트마다 반복해서 적을 필요가 없죠. tags=["todos"]는 Swagger 문서에서 같은 그룹으로 묶어 보여줍니다.

Fast FastAPI

app.include_router(todos.router)

@Module imports ↔ include_router. 더 가볍고 명시적입니다.

🧭 라우터에도 공통 의존성을 걸 수 있다

APIRouter(dependencies=[...])로 그 라우터 전체에 공통 처리(예: 인증)를 걸 수 있습니다. NestJS에서 모듈/컨트롤러 단위로 Guard를 거는 것과 비슷한데, 이건 Week 6의 Depends와 Week 8의 인증에서 본격적으로 활용합니다.

⚠️ prefix 중복 주의

APIRouter(prefix="/todos")를 줬다면 라우트 데코레이터엔 @router.get("") 또는 @router.get("/{id}")처럼 쓰세요. 여기에 또 @router.get("/todos")라고 하면 경로가 /todos/todos가 되어버립니다.

복습

5✅ 셀프 체크

Q APIRouter는 NestJS의 무엇에 대응하나요?

도메인별로 라우트를 묶는다는 점에서 NestJS의 Module + Controller 묶음에 대응합니다. include_router가 @Module의 imports처럼 메인 앱에 라우터를 등록합니다.

Q prefix를 쓰면 좋은 점은?

그 라우터의 모든 경로 앞에 공통 경로가 자동으로 붙어, 라우트마다 /todos를 반복하지 않아도 됩니다. 도메인 경로를 한 곳에서 관리하니 변경도 쉽습니다.

Q 경로가 /todos/todos로 잘못 나옵니다. 원인은?

라우터에 prefix="/todos"를 줬는데 라우트 데코레이터에도 "/todos"를 또 적었기 때문입니다. prefix가 있으면 데코레이터엔 나머지 경로("", "/{id}")만 적어야 합니다.
