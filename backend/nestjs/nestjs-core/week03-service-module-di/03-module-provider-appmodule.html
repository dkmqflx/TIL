AppModule · main.ts

@Module()로 기능을 묶고, exports/imports로 모듈 간 공유를 제어한다. AppModule이 뿌리가 되어 main.ts에서 앱이 시작되는 전체 그림을 본다.

"

모듈은 NestJS 앱의 조립 단위다. 컨트롤러·프로바이더를 묶어 하나의 기능 도메인으로 캡슐화하고, exports/imports로 필요한 것만 공유한다. NestFactory.create(AppModule) 한 줄이 이 모든 것을 깨워 HTTP 서버를 만든다.

  • @Module()의 네 옵션(controllers, providers, imports, exports)을 설명할 수 있다
  • 한 모듈의 프로바이더를 다른 모듈에서 쓰려면 무엇이 필요한지 말할 수 있다
  • 루트 모듈(AppModule)의 역할과 기능 모듈의 차이를 설명할 수 있다
  • main.ts의 부트스트랩 코드가 하는 일을 순서대로 말할 수 있다

1@Module() 구조 — 네 가지 옵션

@Module() 데코레이터는 NestJS에게 "이 클래스는 하나의 기능 단위(모듈)다"라고 알립니다. 네 옵션이 모듈의 조립 방식을 결정합니다.

옵션역할예
controllers이 모듈에서 HTTP 요청을 처리할 컨트롤러 목록[PostsController]
providersDI 컨테이너에 등록할 서비스·프로바이더 목록 (이 모듈 안에서만 접근)[PostsService]
imports다른 모듈을 가져와 그 모듈의 exports를 이 모듈에서 사용 가능하게 함[TypeOrmModule]
exports다른 모듈이 import할 때 사용할 수 있도록 공개할 프로바이더 목록[PostsService]
posts.module.ts
import { Module } from '@nestjs/common';
import { PostsController } from './posts.controller';
import { PostsService } from './posts.service';

@Module({
  controllers: [PostsController],  // HTTP 처리 담당
  providers:   [PostsService],     // DI 컨테이너에 등록
  exports:     [PostsService],     // 다른 모듈에 공개
})
export class PostsModule {}

2providers 등록과 모듈 스코프

프로바이더는 모듈 단위로 등록됩니다. providers에 넣은 서비스는 해당 모듈 안에서만 주입받을 수 있습니다. 다른 모듈이 쓰려면 반드시 exports와 imports가 모두 필요합니다.

⚠️ providers에 넣는 것만으로는 다른 모듈에서 못 씁니다

providers: [PostsService]는 이 모듈 안에서만 주입 가능합니다. 다른 모듈(예: CommentsModule)에서 PostsService를 주입받으려면: ① PostsModule의 exports에 PostsService를 추가하고, ② CommentsModule의 imports에 PostsModule을 추가해야 합니다. 둘 중 하나만 해서는 동작하지 않습니다.

3모듈 간 exports/imports로 공유

실제로 CommentsModule이 PostsService를 사용하는 경우를 봅니다.

모듈 간 공유 — exports + imports
// posts.module.ts — PostsService를 공개
@Module({
  providers: [PostsService],
  exports:   [PostsService],   // ① 공개 선언
})
export class PostsModule {}

// comments.module.ts — PostsModule을 가져와 PostsService를 주입받음
@Module({
  imports:   [PostsModule],    // ② 가져오기
  providers: [CommentsService],
})
export class CommentsModule {}

// comments.service.ts — 이제 PostsService를 주입받을 수 있음
@Injectable()
export class CommentsService {
  constructor(private postsService: PostsService) {}
}
🍔 비유 — 부서 간 재료 공유

포스트팀(PostsModule) 주방에는 특별한 소스(PostsService)가 있습니다. 이 소스를 외부에 공개(exports)하지 않으면 다른 팀이 아무리 요청해도 받을 수 없습니다. 댓글팀(CommentsModule)이 그 소스를 쓰고 싶다면 ① 포스트팀이 소스를 공개(exports)하고, ② 댓글팀이 포스트팀과 공식으로 협력(imports)해야 합니다. 두 조건이 모두 맞아야 소스가 흐릅니다.

4main.ts 부트스트랩 — NestFactory.create(AppModule)

main.ts는 NestJS 앱이 시작되는 진입점입니다. 단 몇 줄로 모듈 트리를 초기화하고, DI 컨테이너를 구성하고, HTTP 서버를 시작합니다.

main.ts — 부트스트랩
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  // 1. AppModule을 루트로 NestJS 앱 인스턴스 생성
  //    → 모듈 트리 탐색 · DI 컨테이너 초기화 · 프로바이더 인스턴스화
  const app = await NestFactory.create(AppModule);

  // 2. 3000 포트에서 HTTP 요청 대기 시작
  await app.listen(3000);
  console.log('Application running on http://localhost:3000');
}

bootstrap();
app.module.ts — 루트 모듈
import { Module } from '@nestjs/common';
import { PostsModule } from './posts/posts.module';

@Module({
  imports: [PostsModule],   // 기능 모듈을 여기에 조립
})
export class AppModule {}
💡 AppModule은 조립만, 로직은 기능 모듈에

AppModule 자체는 컨트롤러나 서비스를 직접 갖지 않는 것이 일반적입니다. 기능 모듈(PostsModule, UsersModule 등)을 imports로 가져와 조립하는 역할만 합니다. 이렇게 하면 기능별로 독립적으로 개발·테스트할 수 있고, AppModule은 앱의 전체 그림을 한 눈에 보여주는 목차 역할을 합니다.

복습

5✅ 셀프 체크 — 면접관처럼 물어보기

Q PostsModule의 PostsService를 CommentsModule에서 쓰려면 어떻게 해야 하나요?

두 단계가 모두 필요합니다. ① PostsModule의 exports 배열에 PostsService를 추가해서 "외부에 공개"합니다. ② CommentsModule의 imports 배열에 PostsModule을 추가해서 "가져옵니다". 그 이후에야 CommentsService의 생성자에서 PostsService를 주입받을 수 있습니다. exports 없이 imports만 하거나, imports 없이 exports만 해도 동작하지 않습니다.

Q AppModule(루트 모듈)의 역할은 무엇인가요?

AppModule은 애플리케이션의 진입점 모듈입니다. NestFactory.create(AppModule)에 넘겨지며, NestJS가 이 모듈을 기점으로 모듈 트리를 탐색하고 DI 컨테이너를 초기화합니다. AppModule은 보통 로직을 직접 갖지 않고, 기능 모듈(PostsModule, UsersModule 등)을 imports로 조립하는 역할만 합니다. 앱 전체의 의존 관계를 한 눈에 볼 수 있는 목차 역할을 합니다.

Q @Module({ providers: [PostsService] })와 @Module({ providers: [PostsService], exports: [PostsService] })의 차이는?

providers만 있으면 이 모듈 내부에서만 PostsService를 주입받을 수 있습니다. exports에 추가하면 다른 모듈이 이 모듈을 import할 때 PostsService를 주입받을 수 있게 공개됩니다. 즉, providers는 "등록", exports는 "공개"입니다. 같은 모듈 안에서만 쓴다면 exports가 필요 없지만, 모듈 경계를 넘어야 한다면 반드시 exports에 선언해야 합니다.
