관계 TypeORM 구현

@OneToOne · @ManyToOne / @OneToMany · @ManyToMany 데코레이터를 사용해 엔티티 관계를 선언하고, FK와 조인 테이블이 어느 쪽에 생성되는지 정확히 이해한다.

"

TypeORM에서 관계는 데코레이터 하나로 선언된다. 그러나 어느 쪽 엔티티에 어떤 데코레이터를 붙이느냐가 FK 컬럼의 위치를 결정한다 — 이 규칙을 틀리면 테이블 구조 자체가 잘못된다.

  • @OneToOne + @JoinColumn을 쌍으로 선언하고 FK 소유 쪽을 정확히 지정할 수 있다
  • @ManyToOne / @OneToMany가 짝임을 이해하고, FK가 ManyToOne 쪽에만 생긴다는 것을 안다
  • @ManyToMany + @JoinTable로 중간 테이블을 자동 생성할 수 있다
  • 각 관계 데코레이터의 두 인수(타입 함수, 역방향 함수)가 하는 역할을 설명할 수 있다

1@OneToOne + @JoinColumn — 1:1 관계와 FK 소유

1:1 관계는 두 엔티티 모두에 @OneToOne을 선언합니다. 단, FK 컬럼을 실제로 갖는 쪽(소유쪽)에만 @JoinColumn을 추가합니다. @JoinColumn 없이 @OneToOne만 있는 반대쪽은 FK 없이 관계만 참조합니다.

🍔 비유 — 여권과 사람

한 사람은 여권 하나만 가질 수 있고, 여권도 주인 한 명뿐입니다. 여권(Profile) 서류에는 "소지인 번호"(FK)가 인쇄되어 있지만, 사람(User) 그 자체에는 여권 번호가 새겨져 있지 않습니다. TypeORM에서 @JoinColumn은 "이 서류가 FK를 보관한다"는 선언입니다.

user.entity.ts + profile.entity.ts — @OneToOne
// user.entity.ts
import { Entity, PrimaryGeneratedColumn, Column, OneToOne } from 'typeorm';
import { Profile } from './profile.entity';

@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  email: string;

  // 반대쪽: @JoinColumn 없음 → FK 컬럼이 생기지 않는다
  @OneToOne(() => Profile, (profile) => profile.user)
  profile: Profile;
}

// profile.entity.ts
import { Entity, PrimaryGeneratedColumn, Column, OneToOne, JoinColumn } from 'typeorm';
import { User } from './user.entity';

@Entity()
export class Profile {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  bio: string;

  // FK 소유쪽: @JoinColumn → profiles 테이블에 userId 컬럼 생성
  @OneToOne(() => User, (user) => user.profile)
  @JoinColumn()
  user: User;
}

2@ManyToOne / @OneToMany — 1:N 관계와 FK는 항상 ManyToOne

1:N 관계에서 TypeORM은 두 개의 데코레이터를 쌍으로 씁니다. N쪽(자식) 엔티티에는 @ManyToOne, 1쪽(부모) 엔티티에는 @OneToMany입니다. FK 컬럼은 언제나 @ManyToOne이 있는 쪽 테이블에만 생성됩니다 — @OneToMany만 있는 쪽에는 어떤 FK 컬럼도 생기지 않습니다.

user.entity.ts + post.entity.ts — @ManyToOne / @OneToMany
// user.entity.ts — 1쪽 (부모)
import { Entity, PrimaryGeneratedColumn, Column, OneToMany } from 'typeorm';
import { Post } from './post.entity';

@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  email: string;

  // @OneToMany: users 테이블에 FK 컬럼이 생기지 않는다
  @OneToMany(() => Post, (post) => post.author)
  posts: Post[];
}

// post.entity.ts — N쪽 (자식), FK가 생기는 쪽
import { Entity, PrimaryGeneratedColumn, Column, ManyToOne } from 'typeorm';
import { User } from './user.entity';

@Entity()
export class Post {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  title: string;

  // @ManyToOne: posts 테이블에 authorId(FK) 컬럼이 생성된다
  @ManyToOne(() => User, (user) => user.posts, { onDelete: 'CASCADE' })
  author: User;
}
⚠️ @OneToMany만 단독으로 선언할 수 없다

@OneToMany는 반드시 반대쪽 엔티티에 @ManyToOne이 정의되어 있어야 작동합니다. TypeORM은 @OneToMany의 두 번째 인수(역방향 함수)로 반대쪽 @ManyToOne 필드를 참조하기 때문입니다. @ManyToOne 없이 @OneToMany만 선언하면 런타임 오류가 발생합니다.

3@ManyToMany + @JoinTable — N:M과 조인 테이블 자동 생성

N:M 관계에서는 양쪽 엔티티 모두 @ManyToMany를 선언합니다. 그리고 소유쪽(owner) 엔티티에만 @JoinTable을 추가합니다. TypeORM은 @JoinTable을 보고 두 FK를 담는 중간 테이블을 자동으로 생성합니다.

post.entity.ts + tag.entity.ts — @ManyToMany
// post.entity.ts — 소유쪽 (owner): @JoinTable 있음
import { Entity, PrimaryGeneratedColumn, Column, ManyToMany, JoinTable } from 'typeorm';
import { Tag } from './tag.entity';

@Entity()
export class Post {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  title: string;

  // 소유쪽: @JoinTable → posts_tags_tag 중간 테이블 자동 생성
  @ManyToMany(() => Tag, (tag) => tag.posts)
  @JoinTable()
  tags: Tag[];
}

// tag.entity.ts — 비소유쪽: @JoinTable 없음
import { Entity, PrimaryGeneratedColumn, Column, ManyToMany } from 'typeorm';
import { Post } from './post.entity';

@Entity()
export class Tag {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  // 비소유쪽: @JoinTable 없음
  @ManyToMany(() => Post, (post) => post.tags)
  posts: Post[];
}
데코레이터붙이는 위치DB에 생기는 것짝 데코레이터
@OneToOne양쪽 모두FK 없음 (JoinColumn 쪽에 FK)@JoinColumn (소유쪽)
@JoinColumnFK 소유쪽만FK 컬럼@OneToOne
@ManyToOneN쪽(자식)FK 컬럼@OneToMany
@OneToMany1쪽(부모)없음@ManyToOne
@ManyToMany양쪽 모두없음 (JoinTable이 중간 테이블)@JoinTable (소유쪽)
@JoinTable소유쪽만중간 테이블 (FK 2개)@ManyToMany
💡 두 인수의 역할 — 타입 함수 · 역방향 함수

모든 관계 데코레이터의 두 인수를 이해하면 오타로 인한 오류를 예방할 수 있습니다. 첫 번째 () => Post는 관계 대상 타입을 지정합니다(순환 참조를 피하기 위해 함수로 감쌈). 두 번째 (post) => post.author는 반대쪽 엔티티에서 이 관계를 가리키는 필드를 지정합니다. TypeORM은 이 정보로 양방향 JOIN을 수행합니다.

복습

4✅ 셀프 체크 — 면접관처럼 물어보기

Q @JoinColumn은 관계의 어느 쪽에 붙이나요?

@JoinColumn은 FK 컬럼을 실제로 소유하는 쪽에만 붙입니다. 1:1 관계에서는 두 엔티티 중 FK를 갖기로 결정한 한쪽에만 선언합니다(보통 "종속적인" 쪽 — 예: User ↔ Profile에서 Profile). @JoinColumn이 없는 반대쪽에는 FK 컬럼이 생기지 않습니다. ManyToOne에서는 @ManyToOne 자체가 암묵적으로 @JoinColumn 역할을 합니다 — 별도로 선언하지 않아도 N쪽 테이블에 FK 컬럼이 생깁니다.

Q @OneToMany가 선언된 쪽에 FK 컬럼이 생기나요?

아니요. FK는 항상 @ManyToOne이 선언된 N쪽 테이블에만 생깁니다. @OneToMany가 선언된 1쪽(부모) 테이블에는 어떤 FK 컬럼도 추가되지 않습니다. @OneToMany는 오직 "반대쪽에 @ManyToOne으로 연결된 컬렉션 필드"를 노출하기 위한 매핑일 뿐입니다. TypeORM이 내부적으로 JOIN을 수행할 때 N쪽의 FK를 참조합니다.

Q @ManyToMany에서 @JoinTable은 어느 쪽에 붙여야 하나요?

@JoinTable은 소유쪽(owner) 엔티티 하나에만 붙입니다. 어느 쪽이 소유쪽인지는 비즈니스 맥락에 따라 결정하되, 중요한 것은 둘 중 정확히 하나에만 선언해야 한다는 것입니다. 양쪽 모두 @JoinTable을 붙이면 중간 테이블이 두 개 생기는 오류가 발생하고, 아무 쪽도 선언하지 않으면 TypeORM이 중간 테이블을 생성하지 않아 관계가 동작하지 않습니다.
