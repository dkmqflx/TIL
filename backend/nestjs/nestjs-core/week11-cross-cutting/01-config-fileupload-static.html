Config · 파일 업로드 · Static
환경 변수부터 URL 제공까지

ConfigModule로 .env를 읽고, Multer로 파일을 서버에 저장하고, ServeStaticModule로 업로드한 파일을 URL로 제공하는 세 단계를 이어붙인다.

"

환경 변수는 코드 밖에, 파일은 서버 디스크에, URL은 정해진 경로로. 세 모듈을 연결하면 이미지 업로드 → 저장 → URL 응답이 한 흐름으로 완성된다.

  • ConfigModule.forRoot({ isGlobal: true })로 전역 환경 변수를 설정하고 ConfigService.get()으로 값을 읽을 수 있다
  • @UseInterceptors(FileInterceptor('image'))와 @UploadedFile()로 멀티파트 파일을 수신한다
  • ServeStaticModule.forRoot({ rootPath, serveRoot })로 업로드 파일을 특정 URL 경로로 제공한다

1ConfigModule — .env 읽기

NestJS 앱에서 포트, DB URL, JWT 시크릿 같은 값은 코드에 하드코딩하면 안 됩니다. @nestjs/config의 ConfigModule을 루트 모듈에 한 번 등록하면 어느 서비스에서나 ConfigService로 꺼낼 수 있습니다.

app.module.ts — ConfigModule 전역 등록
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,   // 모든 모듈에서 imports 없이 사용 가능
      envFilePath: '.env',
    }),
  ],
})
export class AppModule {}
some.service.ts — ConfigService로 값 읽기
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class SomeService {
  constructor(private readonly configService: ConfigService) {}

  getPort(): number {
    return this.configService.get<number>('PORT') ?? 3000;
  }
}
🧭 .env는 절대 커밋하지 않는다

.env에는 DB 비밀번호, JWT 시크릿, AWS 키 같은 민감 정보가 담깁니다. 반드시 .gitignore에 추가하고, 팀원 공유는 별도 보안 채널(Vault, AWS Secrets Manager)을 이용하세요. 레포에 커밋하면 이미 유출입니다.

2파일 업로드 — Multer + FileInterceptor

HTTP multipart/form-data로 전송된 파일은 NestJS 내장 Multer 연동으로 처리합니다. 컨트롤러 메서드에 @UseInterceptors(FileInterceptor('필드명'))을 달고 @UploadedFile()로 파일 메타데이터를 받습니다.

posts.controller.ts — 파일 업로드 엔드포인트
import {
  Controller, Post, UseInterceptors, UploadedFile,
} from '@nestjs/common';
import { FileInterceptor } from '@nestjs/platform-express';
import { diskStorage } from 'multer';
import { extname } from 'path';

@Controller('posts')
export class PostsController {

  @Post('image')
  @UseInterceptors(
    FileInterceptor('image', {
      storage: diskStorage({
        destination: './public/images',
        filename: (req, file, cb) => {
          const unique = Date.now() + '-' + Math.round(Math.random() * 1e9);
          cb(null, unique + extname(file.originalname));
        },
      }),
    }),
  )
  uploadImage(@UploadedFile() file: Express.Multer.File) {
    return { filename: file.filename };
  }
}
💡 여러 파일 업로드 — FilesInterceptor

한 필드에서 여러 파일을 받으려면 FilesInterceptor('images', 10)(최대 10개)를 쓰고 파라미터를 @UploadedFiles()로 변경합니다. 파일 유형 검증은 fileFilter 옵션에서 file.mimetype을 확인하면 됩니다.

3정적 파일 서빙 — ServeStaticModule

Multer가 public/images/에 저장한 파일을 HTTP로 제공하려면 @nestjs/serve-static의 ServeStaticModule을 등록합니다. rootPath는 디스크 경로, serveRoot는 URL 접두사입니다.

🍔 비유 — 사물함과 열람 창구

업로드는 파일을 사물함(public/)에 넣는 것, Static Serving은 열람 창구를 여는 것입니다. 사물함 번호(rootPath)를 알려주고 창구 번호(serveRoot)를 정하면, 누구든 그 창구 URL로 파일을 꺼내 볼 수 있습니다. NestJS 코드 없이 파일 자체가 응답이 됩니다.

app.module.ts — ServeStaticModule 등록
import { ServeStaticModule } from '@nestjs/serve-static';
import { join } from 'path';

@Module({
  imports: [
    ServeStaticModule.forRoot({
      rootPath: join(__dirname, '..', 'public'),  // 디스크 경로
      serveRoot: '/public',                           // URL 접두사
    }),
  ],
})
export class AppModule {}

등록하면 public/images/abc.jpg는 GET /public/images/abc.jpg로 접근 가능합니다. 업로드 응답에서 filename을 반환하면 클라이언트는 /public/images/{filename}으로 이미지를 표시할 수 있습니다.

4✅ 셀프 체크 — 면접관처럼 물어보기

Q .env 값을 NestJS 코드에서 읽는 방법을 설명하세요.

AppModule에 ConfigModule.forRoot({ isGlobal: true })를 등록합니다. isGlobal: true로 설정하면 다른 모듈에서 ConfigModule을 imports하지 않아도 됩니다. 값이 필요한 서비스 생성자에 ConfigService를 주입하고 this.configService.get<타입>('KEY')으로 읽습니다. .env 파일에 KEY=value 형식으로 선언하면 됩니다.

Q 업로드한 파일을 URL로 보이게 하는 설정은 무엇인가요?

@nestjs/serve-static의 ServeStaticModule.forRoot({ rootPath, serveRoot })를 사용합니다. rootPath에 디스크 경로(예: join(__dirname, '..', 'public')), serveRoot에 URL 접두사(예: '/public')를 지정합니다. 그러면 public/images/abc.jpg 파일이 GET /public/images/abc.jpg로 접근 가능해집니다. NestJS 컨트롤러 없이 파일 자체가 응답이 됩니다.

Q FileInterceptor와 FilesInterceptor의 차이는 무엇인가요?

FileInterceptor('fieldName')은 단일 파일 하나를 받고 파라미터에 @UploadedFile()을 씁니다. FilesInterceptor('fieldName', maxCount)는 같은 필드에서 여러 파일을 받고 파라미터에 @UploadedFiles()를 씁니다. 두 경우 모두 @UseInterceptors()로 메서드에 적용합니다.
